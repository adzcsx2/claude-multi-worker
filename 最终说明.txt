===============================================
✅ 最终方案 - 使用窗格模式
===============================================

命令:
  python cms.py claude:ui,coder,test

实际效果:
  - 在一个 WezTerm 窗口中创建多个窗格 (panes)
  - 每个窗格运行一个独立的 Claude 实例
  - 可以同时看到多个实例运行

===============================================
为什么使用窗格而不是标签页？
===============================================

WezTerm CLI 限制:
  ❌ 不支持直接创建标签页
  ✅ 支持 split-pane 创建窗格
  
窗格的优势:
  ✅ 可以同时看到多个 Claude 实例
  ✅ 便于监控所有实例的状态
  ✅ 更适合协作场景

===============================================
窗格操作:
===============================================

切换窗格:
  Ctrl+Shift+方向键    在窗格间切换
  Ctrl+Shift+ArrowKey  Navigate between panes

调整窗格大小:
  Ctrl+Shift+Alt+方向键

关闭窗格:
  Ctrl+Shift+W

===============================================
完整使用流程:
===============================================

1. 在 WezTerm 中运行:
   cd E:\ai_project\claude-multi-starter
   python cms.py claude:ui,coder,test

2. 等待所有窗格创建完成

3. 在另一个终端发送消息:
   bin\send ui "设计登录页面"
   bin\send coder "实现登录API"
   bin\send test "测试登录功能"

4. 在 WezTerm 中:
   - 所有窗格同屏显示
   - 用 Ctrl+Shift+方向键 切换焦点
   - 每个窗格显示对应 Claude 的响应

===============================================
映射文件:
===============================================

创建: .cms_config\tab_mapping.json
格式:
{
  "tabs": {
    "ui": {"pane_id": "0", "role": "UI/UX designer"},
    "coder": {"pane_id": "1", "role": "developer"},
    "test": {"pane_id": "2", "role": "QA engineer"}
  }
}

send 命令使用这个映射找到对应的窗格
